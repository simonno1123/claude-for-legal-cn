## 一、中文 Prompt 压缩规范

- 多用 **Markdown 表格、列表、标签格式**，少用冗长祈使句

- 相同逻辑的多个条件用表格呈现，不用连续 if-then 段落

## 二、法条引述强制规则

### 2.1 引述格式

引述中国法律法规时必须遵守以下规则：

1. **必须使用法律全称或已声明的缩略名 + 具体条文序号**
   - ✅ 正确：《民法典》第585条第2款
   - ✅ 正确：《劳动合同法》第47条
   - ❌ 错误：根据相关法律规定
   - ❌ 错误：按照民法典规定（无条文号）

2. **严禁编造不存在的法条**
   - 如无法确认条文序号，标注 `[法条待查证]` 并停止引述
   - 绝不可为了看起来权威而编造法条号——这是最严重的幻觉错误

3. **区分效力层级**

   | 效力层级 | 制定主体 | 示例 |
   |---|---|---|
   | 法律 | 全国人大及其常委会 | 《民法典》《劳动合同法》 |
   | 行政法规 | 国务院 | 《职工带薪年休假条例》 |
   | 部门规章 | 国务院各部委 | 《劳动合同法实施条例》（非规章但常被归入此层）|
   | 司法解释 | 最高院/最高检 | 法释〔2020〕26号 |
   | 地方性法规 | 省级人大 | 《北京市工资支付规定》 |
   | 规范性文件 | 各级政府/部门 | 劳社部发〔2005〕12号 |

4. **标注时效性**
   - 如引用的法律已被修订，标注 `[已修订 — 查证现行有效版本]`
   - 如引用的司法解释可能已被新解释替代，标注 `[待确认现行有效性]`

5. **司法解释引用格式**
   - 必须包含：全称 + 发布文号
   - 格式：《最高人民法院关于审理XX案件适用法律问题的解释》（法释〔YYYY〕NN号）

### 2.2 Prompt 中的强制嵌入指令

所有 SKILL.md 在涉及法条引用的步骤中，必须包含以下指令块：

```
> **法条引述规范（强制）。** 引述中国法律法规时：(1) 必须标注法律全称/缩略 + 条文序号（条/款/项）；
> (2) 无法确认条文序号时标注 `[法条待查证]` 并停止——严禁编造；
> (3) 区分法律/行政法规/部门规章/司法解释/地方性法规的效力层级；
> (4) 已修订的法律标注 `[已修订 — 查证现行有效版本]`。
```

## 三、来源标注体系

### 3.1 三级可信度标注

| 标注 | 含义 | 验证优先级 | 使用场景 |
|---|---|---|---|
| `[定论]` | 稳定的法律法规条文引用 | 低（仍需核实） | 《民法典》第585条、《劳动合同法》第47条等核心法条 |
| `[待查证]` | 真实但需验证的引用 | **高** | 司法解释、部门规章、行政指导、案例裁判要旨、生效/修订日期 |
| `[精确引用待查证]` | 具体款项号/文号/案号 | **最高** | 法释文号、具体条款的款/项序号、裁判文书案号 |

### 3.2 来源渠道标注

| 标注 | 含义 |
|---|---|
| `[北大法宝]` | 通过北大法宝 MCP 连接器检索获取 |
| `[威科先行]` | 通过威科先行 MCP 连接器检索获取 |
| `[裁判文书网]` | 通过中国裁判文书网检索获取 |
| `[Alpha]` | 通过 Alpha 法律数据库检索获取 |
| `[法信]` | 通过法信平台检索获取 |
| `[网络检索 — 待查证]` | 网络搜索获取，需核实 |
| `[模型知识 — 待查证]` | 来自模型训练数据，需核实 |
| `[用户提供]` | 用户直接提供 |

### 3.3 无研究工具连接时的审阅人备注

当未配置任何法律研究连接器时，审阅人备注中记录：

```
**来源：** 未连接法律研究工具 — 引用来自模型训练知识。以下内容幻觉风险较高，请优先核查：
法条具体款项序号、司法解释文号、裁判案例案号、地方性法规有效性、最新修法动态。
```

## 五、争议解决框架

中国法律实务中，争议解决有两条主路径。所有涉及争议解决条款审查的 Skill 必须区分：

| 路径 | 适用场景 | 核心法律 | 关键差异 |
|---|---|---|---|
| **仲裁** | 商事纠纷、涉外纠纷 | 《仲裁法》 | 一裁终局；需有效仲裁协议；可选择仲裁机构（CIETAC/BAC/各地仲裁委） |
| **诉讼** | 一般民商事纠纷 | 《民诉法》 | 两审终审制；级别管辖 + 地域管辖 + 专属管辖；可约定管辖法院（有限制） |

特殊路径：
- **劳动争议**：仲裁前置（先仲裁后诉讼，《劳动争议调解仲裁法》）
- **行政争议**：行政复议/行政诉讼
- **知识产权**：专门法院管辖（知识产权法院/互联网法院）

## 七、效力层级验证规则（CLAUDE.md 锚定用）

所有模块的 CLAUDE.md 中必须嵌入以下指令，确保模型在引用法律时自动检验效力层级：

```markdown
## 效力层级验证规则（全局强制 — 读取任何子模块前执行）

在引用或分析任何中国法律法规时，必须执行以下验证链：

1. **识别法律渊源**：确定引用的是法律、行政法规、部门规章、司法解释还是地方性法规
2. **确认现行有效性**：检查是否已被修订、废止或替代
3. **检查上位法冲突**：下位法不得与上位法冲突（《立法法》第87-92条）
4. **标注时效性风险**：对可能已过时的引用标注 `[待确认现行有效性]`
5. **地方差异提示**：如涉及地方性规定（如各地产假天数、最低工资），明确标注 `[地方规定 — 以 XX 省/市现行标准为准]`
```

## 八、混合语言规范

- JSON Schema 键名（key）：`counterparty_type`、`risk_level`、`approval_required`
- 目录名和文件名：`skills/`、`agents/`、`SKILL.md`、`CLAUDE.md`
- 代码变量、函数名
- MCP 工具名称和参数名

- JSON Schema 中的 `description` 字段
