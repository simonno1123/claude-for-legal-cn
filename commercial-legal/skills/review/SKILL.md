# /review

## 强制规则

- 先读取商事合同画像；缺失时提示运行 `/commercial-legal:cold-start-interview`。
- 不按英文标题机械路由。必须识别主协议、订单、附件、SLA、DPA、补充协议、报价单、采购单之间的组合。
- 先确认我方身份：采购方、销售方、服务提供方、客户、披露方、接收方。
- 涉外、双语、境外仲裁、数据出境、外汇或出口管制必须标注专项复核。
- 前置读取 `references/china-approval-and-seal-policy.md`、`references/china-vendor-review-checklist.yaml`、`references/china-dispute-resolution.md`，先做用印授权、发票税务、争议解决三项红旗筛查。

## 前置红旗筛查

1. **先票后款和税务损失**
   - 发现“收到发票后付款”“先票后款”“增值税专用发票”时，必须审查合法有效且可抵扣发票是否被明确约定为付款前提条件。
   - 发现虚开、异常凭证、三流不一致、开票主体与签约/收款主体不一致、无法抵扣时，必须标注税务升级，并路由 `vendor-agreement-review` 或 `saas-msa-review`。

2. **用印授权和电子签**
   - 发现项目章、部门章、技术部章、采购部章、分公司章、办事处章、图片章、空白盖章合同、离职员工签约时，必须标注表见代理/萝卜章风险。
   - 发现电子签时，必须要求核验可靠电子签名四要素；CA 证书、实名认证、签署意愿确认、时间戳、签署日志、文件哈希和签署完成证书作为证据增强项审查。

3. **争议解决无效陷阱**
   - 发现“可向法院起诉或提交仲裁”“诉讼或仲裁”“或审或仲”时，必须标注 P0，并要求改成单一路径。
   - 发现“双方均可向各自所在地法院起诉”“原告所在地或被告所在地法院均有管辖权”时，必须标注管辖条款不确定风险，并要求改为一个确定且有实际联系的法院。

## 路由规则

| 命中内容 | 路由 |
|---|---|
| 供应商、采购、服务采购、设备采购、外包 | `vendor-agreement-review` |
| 销售框架、销售服务协议、销售订单、经销/代理销售、客户采购条款 | `vendor-agreement-review`（按我方销售方/服务提供方视角审查） |
| SaaS、云服务、软件许可、API、平台服务、SLA、DPA | `saas-msa-review` |
| NDA、保密协议、商业秘密、技术交流、样品测试 | `nda-review` |
| 自动续约、到期、取消通知、涨价 | `renewal-tracker` |
| 补充协议、变更单、订单历史 | `amendment-history` |
| 非标条款、金额阈值、用印、数据、税务 | `escalation-flagger` |
