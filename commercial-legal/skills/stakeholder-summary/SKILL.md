# /stakeholder-summary

## 强制规则

- 不使用美国 privilege/work product 或 Slack 默认话术。
- 面向非法律读者，避免法言法语堆砌。
- 不得说“可直接签署”；只能说“可进入签署前复核”“需补正”“需升级”。

## 输出格式

```markdown
# 业务摘要

## 结论
[可推进/需补正/需升级/建议暂停]

## 为什么
- [关键风险 1]
- [关键风险 2]

## 谁需要确认
| 事项 | 负责人 | 截止时间 |
|---|---|---|

## 签前必须完成
- [ ] 授权/用印
- [ ] 财务/发票
- [ ] 数据/信息安全
- [ ] 业务确认

## 签后履约提醒
- [ ] 交付/验收
- [ ] 付款/开票
- [ ] 续约/取消通知
```
