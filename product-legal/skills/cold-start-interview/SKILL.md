# /cold-start-interview

## 目标

生成 `~/.claude/plugins/config/claude-for-legal/product-legal/CLAUDE.md`，作为 `product-legal` 的唯一产品合规画像。不得写入境外产品律师、境外广告监管或英美法保密特权默认框架。

`customize` 已并入本技能。地区、行业、平台和企业风险阈值均应写入同一个产品画像，避免多入口配置冲突。

## `--check-integrations`

仅检查：

- 国家法律法规数据库 / 中国人大网；
- 国务院、部委、地方监管官网；
- WPS / 金山文档 / 本地文件；
- 飞书 / 企业微信 / 钉钉（如存在）。

不得把国际办公或境外法律检索工具标记为中国版默认能力。

## 访谈问题

一次最多问 3 个问题。缺失项写 `[PLACEHOLDER]`，不得自行编造。

### 1. 企业和产品

- 公司/产品名称是什么？
- 产品类型：B2C、B2B、平台、自营电商、SaaS、内容社区、硬件、App/小程序、游戏、直播或其他？
- 主要服务地区是否为中国大陆？是否有港澳台或境外投放？

### 2. 用户和交易

- 是否面向消费者、未成年人、老年人、学生、患者、投资者等特殊群体？
- 是否涉及付款、会员、自动续费、预付费、押金、虚拟权益、优惠券、积分或抽奖？
- 是否已有七日无理由退货、三包、售后、发票、客服和投诉处理规则？

### 3. 广告和营销

- 营销渠道：官网、App、短视频、直播、达人种草、信息流广告、短信、社群或线下物料？
- 是否使用绝对化用语、比较宣传、效果承诺、用户案例、客户 logo、排行榜或奖项？
- 是否涉及医疗、药品、保健食品、金融、教育、招商投资、房地产等特殊广告？

### 4. 数据、算法和 AI

- 是否新增个人信息、敏感个人信息、儿童个人信息、委托处理、共享或出境？
- 是否使用画像、推荐排序、自动化决策、差别定价、内容审核、深度合成或生成式 AI？
- 是否已有隐私政策、个人信息处理规则、算法备案、AI 备案或安全评估材料？

### 5. 平台、电商和内容治理

- 产品角色是平台经营者、自营经营者、商家工具、广告服务商还是内容社区？
- 是否有商家入驻、资质审核、评价管理、排序推荐、投诉处理、通知删除、保证金或退款规则？
- 是否存在 UGC、版权/商标投诉、违法有害信息、社区规则或账号封禁？

### 6. 质量、安全和行业监管

- 是否涉及实物商品、硬件、食品、化妆品、药品、医疗器械、儿童用品、电器、交通出行等？
- 是否需要强制性国家标准、CCC、检测报告、警示说明或召回机制？
- 主要主管部门：市监局、网信办、工信部门、文旅、药监、金融监管、教育主管部门或其他？

## 写入模板

使用 `product-legal/CLAUDE.md` 的中国版结构，填充：

- `Who We Are`
- `Available Integrations`
- `Product Launch Process`
- `Review Framework`
- `Risk Calibration`
- `Marketing Claims`
- `Seed Documents`
