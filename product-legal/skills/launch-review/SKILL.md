# /launch-review

## 适用场景

当用户提出“这个功能能不能上线”“帮我审 PRD”“产品发布前做法务审查”“这个活动/价格页/平台规则有没有风险”时，使用本技能。

输出仅为内部法律/合规审查初稿，不作最终上线批准。任何“可上线”“可投放”“可发布”结论必须经中国执业律师或企业法务确认。

## 强制边界

本技能只按中国大陆产品合规框架运行。不得使用境外消费者、广告、隐私、行业监管或普通法特权框架作为默认结论。涉境外投放时，仅提示需当地律师确认。

## Inputs

先收集并列明缺口：

| 信息 | 必要内容 |
|---|---|
| 产品/功能 | 功能描述、用户路径、上线范围、上线时间、目标用户 |
| 用户对象 | B2C/B2B、未成年人、老年人、学生、患者、投资者等敏感人群 |
| 交易模式 | 自营、平台、商家入驻、会员订阅、自动续费、预付费、押金、虚拟权益 |
| 营销与价格 | 广告语、海报、直播脚本、达人种草、价格页、优惠规则、比较宣传 |
| 数据与算法 | 收集信息、处理目的、共享/委托/出境、自动化决策、画像、推荐、AI 生成 |
| 质量与安全 | 产品标准、检测/认证、警示说明、售后、三包、召回、缺陷反馈 |
| 行业监管 | 医疗、药品、保健食品、金融、教育、游戏、直播、电信、出行等 |

## 中国大陆 canonical 上线审查矩阵

| # | 栏目 | 核心检查 |
|---|---|---|
| 1 | 用户权益与消费者保护 | 七日无理由退货、三包、真实全面告知、格式条款、自动续费、预付费、押金、欺诈赔偿 |
| 2 | 电商与平台责任 | 商家资质、自营与非自营区分、搜索结果广告标识、排序推荐、评价治理、搭售、默认勾选 |
| 3 | 网络安全与个人信息门槛 | 最小必要、明示告知、单独同意、第三方 SDK、敏感个人信息、儿童个人信息、数据出境 |
| 4 | 算法与 AI 触点 | 自动化决策、差别定价、算法推荐、深度合成、生成式 AI 标识、备案和内容安全 |
| 5 | 质量标准与召回 | 强制标准、CCC、产品说明书、警示标识、检测报告、缺陷报告、停止销售和召回 |
| 6 | 广告、价格与促销 | 绝对化用语、事实性效果承诺、比较广告、价格促销、划线价、到手价、特殊广告、直播带货 |
| 7 | 监管责任与整改路径 | 市场监管、网信、工信、消协、平台规则、行业主管部门、行政处罚、下架和限期整改 |

质量安全/召回是默认必检栏目。即使是互联网产品，也必须说明“不涉及实物、硬件、强制标准或召回”的理由。

数据、算法和 AI 是最低限度门槛检查。本技能必须先识别并给出初步风险等级；触发高风险后，再转交 `/privacy-legal` 或 `/ai-governance-legal`，不得只写“另行处理”。

## Severity

| 等级 | 含义 |
|---|---|
| Block | 不建议上线，需先补齐法定许可、认证、监护人同意、强制标识或重大缺陷处置 |
| Danger | 高风险，需整改并经法务复核后上线 |
| Needs Review | 需补材料、查地方口径或转专项模块 |
| Conditional Go | 可在列明条件全部完成后上线 |
| Clear | 未发现明显中国法产品合规阻断项 |

以下场景默认不低于 `Danger`：未成年人、敏感个人信息、自动化决策差别待遇、特殊广告、直播带货、质量缺陷、强制认证缺失、面向公众提供生成式 AI 服务。
