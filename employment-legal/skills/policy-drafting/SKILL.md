# /policy-drafting

## 中国法域与引用规则（强制）

- 默认适用中华人民共和国大陆地区劳动法律体系。
- 不得生成美国式 state supplements、PTO、FMLA、final pay 等制度模板。
- 引用中国法律法规时，必须写明法律全称或已声明缩略名 + 条/款/项；无法确认时标注 `[法条待查证]`。
- 涉及劳动者切身利益的规章制度，必须提示履行民主程序和公示告知程序。
- 输出为制度草案，不得直接宣称“已生效”。

## 定位

本技能是“中国企业规章制度/员工手册三部曲”的第一环：

1. `/policy-drafting`：起草单项制度。
2. `/handbook-updates`：把制度变更与旧手册做 diff。
3. `/handbook-audit`：年度或全量审计员工手册。

## 核心卡点

中国法下，用人单位制定、修改或决定直接涉及劳动者切身利益的规章制度或重大事项时，应重点审查：

- **内容合法**：不得违反法律、行政法规、强制性规定或地方最低标准。
- **民主程序**：涉及劳动报酬、工作时间、休息休假、劳动安全卫生、保险福利、职工培训、劳动纪律、劳动定额管理等事项时，应经职工代表大会或全体职工讨论，提出方案和意见，并与工会或职工代表平等协商确定。
- **公示告知**：制度应向劳动者公示或告知，并保留签收、培训、考试、系统确认、邮件送达等证据。

缺少民主程序或公示告知留痕时，不得把制度作为稳定的处分或解除依据。

## 输入材料

优先读取：

- 现有员工手册或相关制度。
- 拟起草制度主题、适用人群、业务背景和管理痛点。
- 企业注册地、主要经营地、员工工作地。
- 工会或职代会情况。
- 既往劳动争议、处分、解除、投诉或审计发现。

## 工作流

### Step 1：界定制度属性

判断制度是否涉及劳动者切身利益：

| 制度类型 | 是否通常触发民主程序 | 关注点 |
|---|---|---|
| 考勤、工时、加班、休假 | 是 | 工时审批、加班费、调休、地方假期 |
| 薪酬、绩效、奖金、提成 | 是 | 计算口径、发放条件、举证材料 |
| 奖惩、纪律、解除依据 | 是 | 违纪等级、严重性、证据和申辩 |
| 报销、差旅、费用审批 | 通常视内容而定 | 不得借制度变相克扣工资 |
| 反舞弊、廉洁、利益冲突 | 通常是 | 调查程序、处分衔接、隐私边界 |
| 信息安全、保密、竞业 | 通常是 | 商业秘密范围、岗位适用性、补偿 |

### Step 2：生成制度正文

制度应使用员工可理解的中文，结构建议：

- 目的。
- 适用范围。
- 定义。
- 管理规则。
- 申请/审批/申诉流程。
- 证据和记录留存。
- 违规处理。
- 与劳动合同、员工手册和其他制度的关系。
- 生效条件和版本管理。

### Step 3：合法性和比例原则审查

逐项检查：

- 是否剥夺或变相限制法定权利。
- 是否单方面加重劳动者责任。
- 是否用罚款、押金、扣证件、克扣工资等高风险措施替代合法管理。
- 违纪处分是否分级，是否给员工陈述申辩机会。
- 是否与劳动合同、集体合同、员工手册或既有制度冲突。
- 是否涉及个人信息处理，是否符合必要、最小、告知和授权要求。

### Step 4：附民主程序材料

每次输出必须在制度正文后附：

1. 民主程序流转排期表。
2. 职工代表/全体职工意见征集表。
3. 工会或职工代表协商记录要点。
4. 公示签收单模板。
5. 培训/考试/系统确认留痕清单。

## 人工确认门

以下情形必须升级法务/律师：

- 制度涉及解除、降薪、调岗、处分、罚款、绩效淘汰。
- 制度改变薪酬、奖金、提成、工时、加班、休假、福利。
- 制度将作为第39条严重违纪解除依据。
- 企业无工会、无职代会或民主程序证据不足。
- 制度涉及监控、定位、录音录像、邮件/聊天记录审查等个人信息处理。
