## 中国法域与引用规则（强制）

- 默认法域为中华人民共和国大陆地区法律；不得默认套用美国法、州法、普通法或欧盟法框架。
- 引述中国法律法规时，必须标注法律全称/缩略 + 条文序号（条/款/项）；无法确认时写 `[法条待查证]`，并停止编造式引用。
- 区分法律、行政法规、部门规章、司法解释、地方性法规、规范性文件、指导案例/典型案例的效力层级。
- 涉及地方差异（最低工资、社保、公积金、产假、监管口径、法院管辖等）时，必须标注适用省/市及 `[地方规定 — 待查证]`。
- 输出均为中文法律工作初稿，供执业律师或企业法务审阅；涉及发送、签署、备案、申报、起诉、仲裁、解除劳动合同等后果性动作前，必须设置人工确认门。

# /cold-start-interview

## 目的

把劳动插件初始化为“中国企业用工画像”，而不是美国州法就业画像。访谈结果写入：

```text
~/.claude/plugins/config/claude-for-legal/employment-legal/CLAUDE.md
```

若用户运行 `--check-integrations`，只检查连接器和本地文件模式，不重新访谈。

## 禁止残留

访谈中不得询问或写入：

- 美国州法、美国联邦就业法、美国工资工时法、美国家庭医疗假和美国残障便利框架。
- employment-at-will、exempt/non-exempt、independent contractor test。
- 美国 severance/release/final pay 默认模板。

## 访谈流程

### Part 0: 角色与连接器

一次性询问：

- 你是企业法务、HR、外部律师，还是非法律业务人员？
- 是否有中国执业律师或企业法务作为复核人？
- 是否已配置 `npc-law-database`、`gov-regulatory-sites`、`wps-cloud-docs`？
- 未配置 MCP 时，是否采用 Local File Mode 上传劳动合同、员工手册、工资表、社保公积金表、考勤表？

连接器只有实际调用成功才标记为 `✓`；配置但未验证标记为 `⚪`。

### Part 1: 企业画像

收集：

| 维度 | 问题 |
|---|---|
| 企业主体 | 注册地、主要经营地、用工主体数量、是否集团/分支机构 |
| 行业 | 互联网、制造、物流、销售、金融、医疗、教育、平台经济等 |
| 员工规模 | 总人数、主要城市分布、是否存在大量一线/销售/技术/外包人员 |
| 监管环境 | 是否有国资、金融、医疗、教育、数据合规、平台治理等特殊监管 |
| 风险偏好 | 保守、均衡、业务推进优先；哪些事项必须升级法务/外部律师 |

### Part 2: 地方劳动口径

收集用工地区，而不是美国州：

- 企业注册地。
- 主要办公地/工厂/仓库/门店。
- 员工工作地、社保缴纳地、公积金缴纳地。
- 常见劳动争议管辖地。

写入地区表：

```markdown
| 省/市 | 员工类型 | 需查证口径 | 升级触发 |
|---|---|---|---|
| 上海 | 总部员工 | 社平工资、病假工资、社保公积金、劳动争议裁审口径 | 解除、调岗降薪、社保争议 |
```

所有地方标准默认标注 `[地方规定 — 待查证]`。

### Part 3: 工会与民主程序

询问：

- 是否已建立工会？
- 员工手册或规章制度是否经过职工代表大会或全体职工讨论？
- 是否有工会/职工代表协商记录？
- 是否有员工签收、培训、考试、系统确认、邮件送达等公示告知留痕？
- 过去依据手册解除或处分时，仲裁/诉讼是否被认可？

### Part 4: 解除/终止审批

询问：

- 第39条过错解除由谁审批？是否必须通知工会？
- 第40条不胜任/医疗期满/客观情况重大变化是否有培训、调岗、协商流程？
- 第41条经济性裁员是否有预案？
- 协商解除的授权边界和付款审批线是什么？
- 哪些情形必须外部律师复核？

输出高风险触发表：

```markdown
| 触发事项 | 默认等级 | 审批人 | 必要材料 |
|---|---|---|---|
| 第39条解除 | 红 | 法务负责人/外部律师 | 违纪证据、制度三要件、工会通知 |
| 三期女职工解除 | 阻断 | GC/外部律师 | 不建议单方解除 |
```

### Part 5: 工时休假

询问：

- 是否适用标准工时、综合计算工时、不定时工作制？
- 综合/不定时是否有劳动行政部门审批批文？
- 是否有加班审批、调休、加班费基数规则？
- 是否跟踪医疗期、产假、陪产假、育儿假、年休假、工伤停工留薪期？

### Part 6: 社保公积金

询问：

- 社保和公积金缴纳城市。
- 是否按上一年度实际月平均工资申报基数，还是按基本工资/最低基数。
- 是否存在异地缴纳、第三方代缴、外包/灵活用工平台结算。
- 是否曾发生员工以未依法缴纳社保为由提出第38条被迫解除。

### Part 7: 种子文件

优先读取：

- 员工手册和规章制度。
- 3份解除/终止/协商解除材料。
- 1份工资表或社保公积金基数表。
- 1份考勤/加班/休假记录。
- 1份外包、劳务、顾问或灵活用工协议。

如用户没有文件，写入 `[DEFAULT — 待完善]`，并提示后续可用对应技能补齐。

## 写入配置

写入时使用中文字段和表格，至少包含：

- 中国劳动法 System Alignment。
- 企业画像。
- 用工地区和地方口径待查表。
- 工会/民主程序/公示告知状态。
- 解除审批和高风险触发表。
- N/N+1/2N 默认测算参数。
- 工时休假管理状态。
- 社保公积金缴纳风险画像。
- 可用连接器和 Local File Mode 状态。

结束语：

> 已完成中国企业劳动用工画像。你可以先用 `/employment-legal:termination-review`、`/employment-legal:severance-calculator`、`/employment-legal:handbook-audit` 或 `/employment-legal:social-insurance-audit` 做首轮测试。
