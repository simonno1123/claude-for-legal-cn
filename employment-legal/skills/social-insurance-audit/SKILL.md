## 中国法域与引用规则（强制）

- 默认法域为中华人民共和国大陆地区法律；不得默认套用美国法、州法、普通法或欧盟法框架。
- 引述中国法律法规时，必须标注法律全称/缩略 + 条文序号（条/款/项）；无法确认时写 `[法条待查证]`，并停止编造式引用。
- 区分法律、行政法规、部门规章、司法解释、地方性法规、规范性文件、指导案例/典型案例的效力层级。
- 涉及地方差异（最低工资、社保、公积金、产假、监管口径、法院管辖等）时，必须标注适用省/市及 `[地方规定 — 待查证]`。
- 输出均为中文法律工作初稿，供执业律师或企业法务审阅；涉及发送、签署、备案、申报、起诉、仲裁、解除劳动合同等后果性动作前，必须设置人工确认门。

# /social-insurance-audit

## 目标

识别企业社保、公积金缴纳中的劳动争议和行政合规风险，尤其是少缴/漏缴、异地缴纳、按最低基数缴纳、第三方代缴、灵活用工/外包错配。

## 必要输入

| 数据 | 用途 |
|---|---|
| 员工工资表 | 判断上一年度实际月平均工资和缴费基数差异 |
| 社保公积金缴纳台账 | 检查缴纳城市、基数、月份、险种 |
| 劳动合同/工作地 | 判断是否异地缴纳或主体错配 |
| 外包/劳务/灵活用工协议 | 判断是否假外包真用工 |
| 员工投诉/离职材料 | 判断第38条被迫解除风险 |

## 审计卡点

### 1. 基数合规

检查企业是否以最低工资、基本工资、固定工资或人为拆分工资作为缴费基数，而非按劳动者上一年度实际月平均工资申报。

必须提示：

- 奖金、津贴、补贴、加班费、提成等是否计入，需按当地社保/公积金口径查证。
- 缴费基数上下限每年调整，必须标注 `[地方规定 — 待查证]`。

### 2. 未缴/少缴/漏缴

识别：

- 入职后未及时缴纳。
- 试用期不缴。
- 按最低基数缴纳。
- 社保和公积金缴纳主体与劳动合同主体不一致。
- 工作地与缴纳地长期不一致。
- 通过第三方代缴规避用人单位义务。

### 3. 第38条被迫解除风险

如企业未依法缴纳社会保险，员工可能主张依据《劳动合同法》第38条解除劳动合同，并要求经济补偿 N。

必须提示：

- 是否支持经济补偿存在地方裁审口径差异，尤其需查证当地规则。
- 上海等地区可能存在特定裁审口径，不得输出全国一刀切结论。
- 公积金争议的劳动仲裁/行政投诉路径需区分 `[地方规定 — 待查证]`。

### 4. 多元用工错配

对外包、劳务、灵活用工、个体工商户、平台结算场景，检查：

- 是否实质存在劳动关系。
- 是否由企业直接管理、考勤、发放固定报酬。
- 是否可能被认定为实际用人单位并承担补缴或连带责任。

## 输出格式

```markdown
# 社保公积金合规审计初稿

> **法律工作初稿 — 需律师/法务复核**
> **法域：** 中国大陆 / [省市]
> **来源：** [用户提供 / 法律数据库 / 模型知识 — 待查证]

## 1. 总体风险
| 项目 | 评级 | 说明 |
|---|---|---|
| 社保基数 | 绿/黄/红 |  |
| 公积金基数 | 绿/黄/红 |  |
| 异地/第三方代缴 | 绿/黄/红 |  |
| 第38条被迫解除 | 绿/黄/红 |  |
| 外包/灵活用工错配 | 绿/黄/红 |  |

## 2. 样本差异表
| 员工/群体 | 实际月均工资 | 申报基数 | 差异 | 风险 |
|---|---:|---:|---:|---|

## 3. 法律后果
- 补缴/行政投诉风险
- 第38条被迫解除及 N 风险
- 群体性争议风险
- 外包/灵活用工穿透风险

## 4. 整改建议
1. 核验地方基数上下限。
2. 分层测算补缴敞口。
3. 优先整改高风险人群和存量投诉。
4. 对外包/灵活用工做事实劳动关系复核。
5. 律师/法务确认沟通口径后再对员工发布。
```

## 禁止事项

- 不得承诺“补缴即可完全消除劳动争议风险”。
- 不得忽略地方裁审口径差异。
- 不得建议通过第三方代缴、个体户拆分或假外包规避法定义务。
