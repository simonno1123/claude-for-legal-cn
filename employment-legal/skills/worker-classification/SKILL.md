## 中国法域与引用规则（强制）

- 默认法域为中华人民共和国大陆地区法律；不得默认套用美国法、州法、普通法或欧盟法框架。
- 引述中国法律法规时，必须标注法律全称/缩略 + 条文序号（条/款/项）；无法确认时写 `[法条待查证]`，并停止编造式引用。
- 区分法律、行政法规、部门规章、司法解释、地方性法规、规范性文件、指导案例/典型案例的效力层级。
- 涉及地方差异（最低工资、社保、公积金、产假、监管口径、法院管辖等）时，必须标注适用省/市及 `[地方规定 — 待查证]`。
- 输出均为中文法律工作初稿，供执业律师或企业法务审阅；涉及发送、签署、备案、申报、起诉、仲裁、解除劳动合同等后果性动作前，必须设置人工确认门。

# /worker-classification

## 适用场景

审查企业拟采用或已经采用的多元化用工安排是否可能被认定为劳动关系，尤其是名义上签署《劳务协议》《顾问协议》《外包协议》《承揽合同》《灵活用工协议》，但实质上存在管理与被管理关系的场景。

本技能彻底停用美国 independent contractor test、ABC test、common-law control test 等模型；默认按中国事实劳动关系识别逻辑输出。

## 核心依据

以《关于确立劳动关系有关事项的通知》（劳社部发〔2005〕12号）中的实质要件为核心审查框架，并结合《劳动合同法》、地方仲裁/法院口径和用户材料进行风险判断。

如无法核验文号或现行有效性，标注 `[精确引用待查证]`，但不得改用美国法测试。

## 输入清单

| 类别 | 问题 |
|---|---|
| 主体资格 | 企业是否为合法用工主体？个人是否具备劳动者主体资格？ |
| 管理关系 | 是否接受考勤、排班、绩效、规章制度、奖惩、汇报线管理？ |
| 报酬 | 是否按月/固定周期发放固定报酬？是否类似工资条？是否代扣个税？ |
| 工作内容 | 是否从事企业主营业务、核心岗位、持续性工作？ |
| 工具和场所 | 是否使用企业办公场所、账号、设备、系统、邮箱、工牌？ |
| 排他性 | 是否长期、稳定、排他为企业工作？ |
| 协议文本 | 名义合同是劳动、劳务、外包、承揽、派遣、实习还是灵活用工？ |
| 社保税务 | 是否缴纳社保公积金？是否走劳务报酬、个体工商户、平台结算？ |

## 分类路径

| 类型 | 典型特征 | 主要风险 |
|---|---|---|
| 劳动关系 | 管理从属性强、持续用工、按月报酬、核心业务 | 未签劳动合同双倍工资、社保补缴、解除限制 |
| 劳务关系 | 一次性或较弱管理、结果或服务导向、独立性较强 | 仍可能因事实管理被穿透 |
| 外包/承揽 | 交付成果、承揽方自主管理人员和工具 | 假外包真用工、混同管理 |
| 劳务派遣 | 派遣单位签劳动合同，用工单位实际用工 | 临时性、辅助性、替代性岗位限制及连带责任 |
| 非全日制 | 小时计酬、工作时间较短、可口头协议 | 工时上限、工资结算周期、社保地方口径 |
| 实习/见习 | 教育培养属性、学校/实习协议 | 毕业后继续用工、实质劳动关系风险 |
| 灵活用工/平台用工 | 平台结算、个体工商户、众包 | 核心业务嵌入、算法/考核管理导致从属性 |

## 实质劳动关系红旗

命中越多，越接近事实劳动关系：

- 企业直接安排工作时间、地点、任务和汇报线。
- 要求打卡、请假、绩效考核、遵守员工手册。
- 按月发固定报酬，报酬不以独立成果交付为核心。
- 长期、稳定、排他地为企业提供劳动。
- 使用企业工牌、邮箱、系统账号、办公设备或办公场所。
- 从事企业主营业务或核心岗位，而非独立外部服务。
- 企业对其进行奖惩、培训、晋升、淘汰或纪律处分。
- 名义合同和实际履行不一致。

## 风险后果

如存在事实劳动关系风险，必须提示：

- 未签书面劳动合同双倍工资风险（通常最多 11 个月，需核验时效和地方口径）。
- 社保、公积金补缴及滞纳/行政风险。
- 解除关系时可能适用《劳动合同法》的法定解除和经济补偿/赔偿规则。
- 工伤、加班、年休假、最低工资、医疗期等劳动权益追溯风险。
- 批量同类安排可能引发群体性争议或监管关注。

## 后果性动作门

如用户要求“直接改协议”“解除合作”“补签劳动合同”“停止缴社保”或“批量转外包”，必须提示：

> 该动作可能改变用工关系和劳动争议风险。请确认中国执业律师或企业法务已复核事实履行、地方口径和存量风险。是否继续生成内部讨论草稿？
