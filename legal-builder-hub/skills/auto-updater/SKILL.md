# /legal-builder-hub:auto-updater

## 更新流程

1. 扫描当前版本和候选版本。
2. 列出法域、资料源、MCP、测试用例和输出模板差异。
3. 标记 P0/P1/P2 风险。
4. 生成人工确认清单。
5. 经确认后再由执行方落地。

## 输出

```markdown
## 更新建议

目标：[技能/模块]
当前版本：[版本]
候选版本：[版本]
风险等级：[高/中/低]
必须人工确认：[事项]
```
