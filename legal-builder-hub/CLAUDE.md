# Legal Builder Hub - 中国法律技能构建中心

## 定位

本模块用于构建、审查和维护 `claude-for-legal-cn` 的中国法律技能、插件和 MCP 连接器占位配置。它不是法律意见输出模块，而是工具治理模块。

## 强制规则

- 默认法域为中华人民共和国大陆地区。
- 不得默认引入美国法、普通法、英美证据特权、美国程序法、美国律师协会规范、美国引注规范或境外法律数据库框架。
- 所有新技能必须包含中国法边界、资料来源、人工确认门、回归测试和无凭证 MCP 占位说明。
- 不得在插件或技能中写入 API Key、Token、Cookie、账号密码或商业数据库凭证。
- 所有输出均为“内部工具构建建议 / 技术与合规审查初稿”。

## 中国法技能验收门

一个技能标记为可用前，必须满足：

1. `SKILL.md` frontmatter 有 `name` 和 `description`。
2. 用户可见文案为中文或中英双语，且中国法为默认。
3. 明确列出适用法律、监管机关、资料来源和待核查事项。
4. 不能承诺证据特权、披露豁免或自动合规。
5. 有 `references/test-cases-cn.md` 或模块级回归用例。
6. JSON 可解析，UTF-8 无乱码。
7. MCP 配置只使用环境变量占位符，不包含真实密钥。

## 默认资料源

- 国家法律法规数据库、中国人大网
- 最高人民法院、最高人民检察院
- 国务院、司法部、市场监管总局、网信办、工信部、人社部、证监会等主管机关
- 地方人大、地方政府、地方监管机关
- 企业上传的合同、制度、案件、审批和内控材料

## 常用工作流

1. `registry-browser`：查看可用技能和本地化状态。
2. `skills-qa`：审查技能是否符合中国法本土化标准。
3. `skill-installer`：安装或接入通过审查的技能。
4. `skill-manager`：管理启用、禁用、更新和回滚。
5. `auto-updater`：生成更新计划，但不自动覆盖用户改动。
