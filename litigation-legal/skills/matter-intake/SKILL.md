# /matter-intake

## 强制规则

- 调用 `references/china-litigation-core-rules.md`。
- 不使用 complaint-served/subpoena/discovery 等美国案件来源分类；改用中国程序来源。
- 不直接建议起诉或仲裁；只能输出“可进入律师复核/需补证/需先保全/需重新判断路径”。
- 将机器辅助生成的 Issue、Question、Claim 和 Request Right 均标记为 `candidate`；未经具备相应资格的律师复核，不得用于起诉、仲裁、保全、发函、谈判或法律意见。
- 保留来源、现行有效性、法域、时间适用性、不利事实、不确定性和信息缺口；不得自动选择请求权基础或认定其可用。

## 收案审查顺序

1. **程序路径**
   - 民事诉讼、商事仲裁、劳动仲裁、诉前调解、行政/监管、执行。
   - 是否存在有效仲裁条款；是否存在或审或仲、仲裁机构名称不明、法院管辖约定不确定。

2. **主体资格**
   - 原告/被告/第三人/申请人/被申请人。
   - 自然人身份、企业统一社会信用代码、法定代表人、分支机构、关联公司、共同被告、追加主体。

3. **管辖**
   - 级别管辖、地域管辖、专属管辖、协议管辖。
   - 合同履行地、被告住所地、不动产所在地、侵权行为地。

4. **诉讼时效和程序期限**
   - 普通三年时效、起算点、中止/中断线索、最长保护期。
   - 答辩、举证、上诉、申请执行等期限如已出现文书则单独列明。

5. **诉请和请求权基础**
   - 每项诉请对应事实、法律依据、金额计算、利息/违约金、律师费/保全费/诉讼费承担。
   - 区分委托人目标、请求的救济、主张的 Claim 和候选 Request Right；逐项记录依据来源、适用法域、现行有效性、时间适用性、不利条件和待核验问题。

6. **证据初筛**
   - 按证据编号输出真实性、合法性、关联性、证明目的。
   - 识别缺原件、电子数据无原始载体、录音合法性不足、发票/流水不闭环等风险。

7. **保全和调查令**
   - 财产保全：金额、财产线索、担保、紧迫性。
   - 证据保全：可能灭失或以后难以取得。
   - 行为保全：禁令/维持现状。
   - 调查令：银行流水、工商内档、平台实名信息、社保公积金、不动产登记等。

8. **送达和执行**
   - 被告送达地址、电子送达可能性、公告送达风险。
   - 被告履行能力、财产线索、执行难度、和解可执行性。
