# /dpa-review

## 强制规则

- 不得把 controller/processor/SCC 作为默认中国法结构；遇到英文 DPA 时，先映射为个人信息处理者、受托处理、共同处理、对外提供或跨境提供。
- 调用 `references/china-privacy-data-playbook.md`。
- 输出不得写“可签署”，只能写“可进入签署前复核/需补正/需升级”。

## 审查类型

1. **受托处理**
   - 处理目的、期限、方式、个人信息种类、保护措施、双方权利义务、返还/删除、转委托、审计和违约责任。

2. **共同处理**
   - 各方目的、权利义务、个人权利响应、对外告知、责任分担和内部追偿。

3. **向其他个人信息处理者提供**
   - 接收方名称/联系方式、处理目的/方式、个人信息种类、单独同意、接收方保护能力。

4. **跨境提供**
   - 境外接收方、国家/地区、数据类型、规模、目的、再转移、出境路径、安全评估/标准合同/认证或豁免 `[待查证]`。

## 输出格式

```markdown
# 数据处理协议审查初稿

> 来源：
> 协议类型：
> 法域：中国大陆 / [涉外因素]
> 待补材料：

## 结论
[可进入签署前复核 / 需补正 / 需升级 / 暂不建议推进]

## 风险矩阵
| 风险等级 | 条款 | 问题 | 中国法卡点 | 修改建议 |
|---|---|---|---|---|

## 必改条款
- [ ] 处理目的、期限、方式和数据种类
- [ ] 安全措施与事件通知
- [ ] 转委托/第三方提供
- [ ] 返还、删除和证明
- [ ] 审计与配合监管
- [ ] 跨境提供路径
```
