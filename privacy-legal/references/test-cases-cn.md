# 中国个人信息与数据合规 Prompt 回归测试用例

| Case 编号 | 场景标签 | 输入测试数据点 | 预期输出 |
|---|---|---|---|
| Case 1 | 人脸考勤 | 企业拟用人脸识别替代打卡，员工不同意则无法考勤。 | Critical：识别生物识别敏感个人信息；审查必要性、替代方案、单独同意、PIPIA、存储期限和安全措施，不得把“员工同意”当成唯一依据。 |
| Case 2 | App 过度索权 | 手电筒类 App 首次打开即索取通讯录、精确定位和相册权限。 | Danger：最小必要和目的限定不满足；要求按功能触发、逐项告知、非必要权限不得捆绑授权。 |
| Case 3 | SDK 隐蔽收集 | 新接入广告 SDK 收集设备标识、位置信息和剪贴板，隐私政策未披露。 | Critical：识别第三方 SDK 与对外提供风险；要求 SDK 清单、数据流、单独同意/PIPIA、隐私政策同步和下架或整改。 |
| Case 4 | HR 数据出境 | 中国员工绩效、薪酬、身份证和紧急联系人信息同步到境外集团 HR 系统。 | High：识别员工个人信息和跨境提供；要求出境路径、安全评估/标准合同/认证或豁免核验、单独告知同意、最小化和境外接收方约束。 |
| Case 5 | 受托客服外包 | 企业把客户工单、电话录音和订单信息交给外包客服公司处理，协议只写“保密”。 | High：识别受托处理协议不足；要求目的、期限、方式、数据种类、保护措施、转委托、返还删除、审计和违约责任。 |
| Case 6 | 共同营销 | 两家公司联合办活动，报名页同时收集双方营销授权，但未说明各自处理目的。 | High：识别共同处理/对外提供边界；要求明确各方身份、目的、信息种类、权利响应责任和单独同意。 |
| Case 7 | AI 训练客户日志 | SaaS 服务商拟用客户聊天记录和工单日志训练模型，只在服务条款中写“改进服务”。 | Critical：识别二次利用、敏感信息、商业秘密和个人信息风险；要求明确授权、脱敏/匿名化、退出机制、PIPIA、第三方模型边界。 |
| Case 8 | 自动化决策差别定价 | 平台根据消费能力和浏览记录给不同用户展示不同价格。 | Critical：识别自动化决策和不合理差别待遇风险；要求透明说明、拒绝/解释渠道、人工干预、PIPIA 和公平性复核。 |
| Case 9 | 未成年人信息 | 教育 App 收集未成年人的姓名、学校、班级、定位和学习记录。 | Critical：识别未成年人个人信息与敏感场景；要求监护人同意、专门规则、最小化、权限控制和删除机制。 |
| Case 10 | 删除请求 | 用户要求删除账号和全部个人信息，业务称“为经营分析必须永久保留”。 | Warning/High：要求核查法定留存、合同履行和争议保存边界；可删除的应删除或匿名化，不得以经营分析笼统拒绝。 |
| Case 11 | 泄露事件 | 运维误将包含 5 万名用户手机号和订单地址的文件上传公开仓库 12 小时。 | Critical：触发安全事件响应；要求止损、影响评估、证据保全、通知个人/监管报告 `[待查证]`、第三方追责和整改复盘。 |
| Case 12 | 隐私政策与实际不一致 | 隐私政策称“不向第三方共享”，实际向支付、广告和数据分析 SDK 传输用户标识。 | Critical：识别对外承诺漂移和虚假/误导性告知；要求更新政策、SDK 清单、同意链路和停止未披露传输。 |
